---
title: "Quick Start"
description: "Add ShadowHook to your Android project and perform your first inline hook in minutes."
---

<Tip>
  Use unique mode first when getting started. Switch to shared or multi mode once you understand the multi-hook semantics.
</Tip>

<Steps>

<Step title="Add dependency in build.gradle">

ShadowHook is published on [Maven Central](https://central.sonatype.com/artifact/com.bytedance.android/shadowhook). It uses the [Prefab](https://google.github.io/prefab/) package format for native dependencies, supported from Android Gradle Plugin 4.0.

Add the Maven Central repository if it isn't already present:

```gradle build.gradle
allprojects {
    repositories {
        mavenCentral()
    }
}
```

Enable Prefab and add the implementation dependency:

```gradle build.gradle
android {
    buildFeatures {
        prefab true
    }
}

dependencies {
    implementation 'com.bytedance.android:shadowhook:x.y.z'
}
```

Replace `x.y.z` with the latest [release version](https://github.com/bytedance/android-inline-hook/releases).

<Note>
  ShadowHook uses [prefab package schema v2](https://github.com/google/prefab/releases/tag/v2.0.0), which is the default from Android Gradle Plugin 7.1.0. If you are using an earlier AGP version, add the following to `gradle.properties`:

  ```
  android.prefabVersion=2.0.0
  ```
</Note>

</Step>

<Step title="Add dependency in CMakeLists.txt or Android.mk">

<CodeGroup>

```cmake CMakeLists.txt
find_package(shadowhook REQUIRED CONFIG)

add_library(mylib SHARED mylib.c)
target_link_libraries(mylib shadowhook::shadowhook)
```

```makefile Android.mk
include $(CLEAR_VARS)
LOCAL_MODULE           := mylib
LOCAL_SRC_FILES        := mylib.c
LOCAL_SHARED_LIBRARIES += shadowhook
include $(BUILD_SHARED_LIBRARY)

$(call import-module,prefab/shadowhook)
```

</CodeGroup>

</Step>

<Step title="Specify ABIs">

```gradle build.gradle
android {
    defaultConfig {
        ndk {
            abiFilters 'armeabi-v7a', 'arm64-v8a'
        }
    }
}
```

</Step>

<Step title="Add packaging options">

ShadowHook includes two `.so` files: `libshadowhook.so` and `libshadowhook_nothing.so`. The correct packaging option depends on whether you are building an SDK or an app.

<CodeGroup>

```gradle SDK project
android {
    packagingOptions {
        exclude '**/libshadowhook.so'
        exclude '**/libshadowhook_nothing.so'
    }
}
```

```gradle APP project
android {
    packagingOptions {
        pickFirst '**/libshadowhook.so'
        pickFirst '**/libshadowhook_nothing.so'
    }
}
```

</CodeGroup>

**SDK project**: Exclude the `.so` files from your AAR to prevent duplicate `.so` conflicts when the app is packaged.

**APP project**: Use `pickFirst` to resolve conflicts from duplicate `.so` files. Note that this may cause the app to use an incorrect version of ShadowHook if multiple versions are present.

</Step>

<Step title="Initialize ShadowHook">

You must initialize ShadowHook once before calling any of its APIs. You can do this from Java or from native code — choose one.

<Tabs>

<Tab title="Java">

```java MySdk.java
import com.bytedance.shadowhook.ShadowHook;

public class MySdk {
    public void init() {
        ShadowHook.init(new ShadowHook.ConfigBuilder()
            .setMode(ShadowHook.Mode.UNIQUE)
            .build());
    }
}
```

</Tab>

<Tab title="Native (C)">

```c mylib.c
#include "shadowhook.h"

void init() {
    shadowhook_init(SHADOWHOOK_MODE_UNIQUE, false);
}
```

</Tab>

</Tabs>

ShadowHook supports three modes: `shared`, `multi`, and `unique`. Start with `UNIQUE` — it is the simplest to reason about. See [Hook modes](/concepts/hook-modes) for a full comparison.

</Step>

<Step title="Perform your first hook">

Hook applies to an entire function. You write a proxy function with the same signature as the original. When the hooked function is called, your proxy runs first.

The example below hooks `art::ArtMethod::Invoke` in `libart.so`:

```c mylib.c
#include "shadowhook.h"

void *orig = NULL;
void *stub = NULL;

// Type definition of the hooked function
typedef void (*artmethod_invoke_func_type_t)(void *, void *, uint32_t *, uint32_t, void *, const char *);

// Proxy function
void artmethod_invoke_proxy(void *thiz, void *thread, uint32_t *args, uint32_t args_size, void *result, const char *shorty) {
    // do something
    ((artmethod_invoke_func_type_t)orig)(thiz, thread, args, args_size, result, shorty);
    // do something
}

void do_hook() {
    stub = shadowhook_hook_sym_name(
               "libart.so",
               "_ZN3art9ArtMethod6InvokeEPNS_6ThreadEPjjPNS_6JValueEPKc",
               (void *)artmethod_invoke_proxy,
               (void **)&orig);

    if(stub == NULL) {
        int err_num = shadowhook_get_errno();
        const char *err_msg = shadowhook_to_errmsg(err_num);
        LOG("hook error %d - %s", err_num, err_msg);
    }
}

void do_unhook() {
    int result = shadowhook_unhook(stub);

    if(result != 0) {
        int err_num = shadowhook_get_errno();
        const char *err_msg = shadowhook_to_errmsg(err_num);
        LOG("unhook error %d - %s", err_num, err_msg);
    }
}
```

`_ZN3art9ArtMethod6InvokeEPNS_6ThreadEPjjPNS_6JValueEPKc` is the mangled C++ symbol name for `art::ArtMethod::Invoke`. You can find it with `readelf`. Note that the symbol name differs on Android versions before M — handle version differences yourself if you need broader compatibility.

</Step>

<Step title="Optionally intercept">

Intercept applies to individual instructions rather than whole functions. Your interceptor receives all register values and can read or modify them before the instruction continues executing.

The example below intercepts an instruction inside `art::ArtMethod::Invoke` on aarch64:

```c mylib.c
#include "shadowhook.h"

void *stub;

#if defined(__aarch64__)
void artmethod_invoke_interceptor(shadowhook_cpu_context_t *ctx, void *data) {
    // When x19 equals 0, modify the values of x20 and x21
    if (ctx->regs[19] == 0) {
        ctx->regs[20] = 1;
        ctx->regs[21] = 1000;
        LOG("interceptor: found x19 == 0");
    }

    // When q0 equals 0, modify the values of q0, q1, q2, q3
    if (ctx->vregs[0].q == 0) {
        ctx->vregs[0].q = 1;
        ctx->vregs[1].q = 0;
        ctx->vregs[2].q = 0;
        ctx->vregs[3].q = 0;
        LOG("interceptor: found q0 == 0");
    }
}

void do_intercept(void) {
    // Query the address of art::ArtMethod::Invoke
    void *handle = shadowhook_dlopen("libart.so");
    if (handle == NULL) {
        LOG("handle not found");
        return;
    }
    void *sym_addr = shadowhook_dlsym(handle, "_ZN3art9ArtMethod6InvokeEPNS_6ThreadEPjjPNS_6JValueEPKc");
    shadowhook_dlclose(handle);
    if (sym_addr == NULL) {
        LOG("symbol not found");
        return;
    }

    // Locate the address of a certain instruction in art::ArtMethod::Invoke
    void *instr_addr = (void *)((uintptr_t)sym_addr + 20);

    stub = shadowhook_intercept_instr_addr(
               instr_addr,
               artmethod_invoke_interceptor,
               NULL,
               SHADOWHOOK_INTERCEPT_WITH_FPSIMD_READ_WRITE);

    if(stub == NULL) {
        int err_num = shadowhook_get_errno();
        const char *err_msg = shadowhook_to_errmsg(err_num);
        LOG("intercept failed: %d - %s", err_num, err_msg);
    }
}

void do_unintercept() {
    int result = shadowhook_unintercept(stub);

    if (result != 0) {
        int err_num = shadowhook_get_errno();
        const char *err_msg = shadowhook_to_errmsg(err_num);
        LOG("unintercept failed: %d - %s", err_num, err_msg);
    }
}
#endif
```

In this example `instr_addr` is fixed at `sym_addr + 20` for brevity. In real scenarios, use memory scanning or other methods to locate the exact instruction address you want to intercept. Because aarch32 and aarch64 have different registers and instruction layouts, intercept logic generally needs to be written separately for each architecture.

</Step>

</Steps>
